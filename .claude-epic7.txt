## Story 7.3: Assembly Templates

Create src/lib/assembly-templates.ts:
Define 4 built-in templates as constants:
- Essay: slots = [Introduction, Thesis, Evidence 1, Evidence 2, Counterargument, Conclusion]
- Report: slots = [Executive Summary, Background, Findings, Recommendations, Conclusion]
- Decision Brief: slots = [Context, Options, Pros/Cons, Recommendation, Next Steps]

## Story 7.6: Multi-Format Export

Create src/server/services/exportService.ts:
- exportAssembly(assemblyId, format: 'essay'|'presentation'|'email'|'social', db):
  - Fetch assembly with units in order
  - Apply format-specific conversion rules per unit type:
    Essay: claim→thesis paragraph, evidence→supporting paragraph, question→rhetorical heading
    Presentation: each unit→bullet point or slide, type determines formatting
    Email: claim→key point, action→action item, concise
    Social: condensed, 280 char limit per unit
  - Include bridge texts if provided
  - Returns: { content: string, format, unitCount, exportedAt }

Add assembly.export tRPC procedure: { assemblyId, format }

## Story 7.7: Partial Export & Export History

Add ExportHistory model to prisma/schema.prisma if not exists:
model ExportHistory {
  id         String   @id @default(cuid())
  assemblyId String   @map("assembly_id")
  format     String
  unitIds    String[] @map("unit_ids")
  contentHash String  @map("content_hash")
  exportedAt DateTime @default(now()) @map("exported_at")
  assembly   Assembly @relation(fields: [assemblyId], references: [id], onDelete: Cascade)

}
