# Story 3.4: Context View â€” Filtered Unit Display

Status: ready-for-dev

## Story

As a user,
I want to see only the Units belonging to a specific Context when I select it,
So that I can focus on one exploration at a time without distraction.

## Acceptance Criteria

1. **Given** a Context is selected in the sidebar, **When** the main content area updates, **Then** only Units that are members of the selected Context are displayed per FR49
2. Each Unit card shows its perspective-specific type override (if set), importance, and stance for this Context
3. The toolbar breadcrumb updates to show the Context name per UX-DR44
4. Units can be added to the current Context via a dropdown or drag from a global unit list
5. Units can be removed from the current Context (not deleted globally) via the context menu
6. The "All Units" view (no Context filter) remains accessible via a sidebar option
7. The Context's `snapshot_summary` is displayed at the top of the view (placeholder text for now â€” AI generation in Epic 5)

## Dev Notes

- This story connects the sidebar (Story 3.3) to the main content area. The `activeContextId` in the sidebar store is the bridge.
- The filtered query should be efficient â€” use the `unit_context` join table with proper indexes rather than fetching all units and filtering client-side.
- Perspective-aware rendering depends on Story 3.2's `usePerspective` hook. When no perspective override exists, the card shows global defaults seamlessly.
- The "Add Unit" search popover should exclude units already in the context to avoid confusion.
- The snapshot summary is a placeholder for now â€” Epic 5 will generate AI summaries. Just show the field if it has content, or a placeholder message if empty.
