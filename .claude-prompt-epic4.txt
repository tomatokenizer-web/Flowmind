## Story 4.1: Relation CRUD API

Check prisma/schema.prisma - Relation model likely already exists. Check src/server/api/routers/ for existing relation router.

If no relation router exists, create src/server/api/routers/relation.ts with these procedures (all protectedProcedure):
- relation.create: input { sourceUnitId, targetUnitId, perspectiveId?, type: string, strength: z.number().min(0).max(1), direction: 'one_way'|'bidirectional', purpose: string[] }. Before creating, load both units and verify neither has lifecycle='draft' else throw TRPCError BAD_REQUEST.
- relation.update: input { id, strength?, type?, direction?, purpose? }
- relation.delete: input { id }
- relation.listByUnit: input { unitId, contextId? }, returns relations where unitId is sourceUnitId OR targetUnitId, sorted by strength desc
- relation.listBetween: input { sourceUnitId, targetUnitId }

Create src/server/services/relationService.ts with business logic.
Register router in src/server/api/root.ts if not already there.

## Story 4.2: Seed 23 System Relation Types

Check prisma/schema.prisma for SystemRelationType model. Check if prisma/seed.ts exists.

Create or update prisma/seed.ts to upsert all 23 relation types:

Argument (8): supports, contradicts, derives_from, expands, references, exemplifies, defines, questions
Creative/Research (7): inspires, echoes, transforms_into, foreshadows, parallels, contextualizes, operationalizes
Structure/Containment (8): contains, presupposes, defined_by, grounded_in, instantiates, refutes, specifies, abstracts

Each entry: name, category ('argument'|'creative_research'|'structure_containment'), description (1 sentence), sortOrder (int).
Use upsert with where: { name } so re-runs are safe.

Add relationType.list tRPC procedure in src/server/api/routers/relation-type.ts returning all types grouped by category. Register in root.ts.
